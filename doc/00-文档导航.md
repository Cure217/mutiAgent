## 文档清单

- `01-项目目录结构设计.md`
  - 给出推荐的整体目录拆分和前后端协作边界
- `02-后端模块与包结构设计.md`
  - 给出后端模块职责、包结构、适配器设计和会话运行时模型
- `03-MVP接口清单.md`
  - 给出第一阶段应落地的 REST / WebSocket 接口建议
- `04-数据库SQL草案.sql`
  - 给出 SQLite 建表、索引、全文检索和初始化参数草案
- `05-初始化脚手架任务清单.md`
  - 给出第一轮脚手架和实施顺序，便于直接开工
- `06-开发过程记录.md`
  - 记录分阶段开发进度、已完成能力、验证情况和下一步计划

## 推荐先做的事情

1. 先按 `05-初始化脚手架任务清单.md` 调整后端依赖
2. 再按 `01-项目目录结构设计.md` 初始化前端与后端目录
3. 然后执行 `04-数据库SQL草案.sql` 建本地库
4. 最后按 `03-MVP接口清单.md` 开始实现 MVP

## 当前建议基线

- 桌面容器：`Electron`
- 前端：`Vue 3 + TypeScript + Vite + Pinia + Vue Router + Element Plus + xterm.js`
- 后端：`Spring Boot 3.5.x + MyBatis + SQLite + WebSocket + pty4j`
- 运行模式：
  - MVP：`外部终端托管 + 运行状态追踪 + 原始输出采集`
  - 第二阶段：`页内 PTY 嵌入终端 + 结构化消息流`

## 重要提醒

- Windows 本机作为桌面宿主没有问题，但 `Codex`、`Claude Code` 的稳定运行环境优先建议标准化到 `WSL2`
- 本系统不应一开始就假设所有 AI CLI 都能输出标准化 JSON
- 所以数据库必须同时保存：
  - 结构化消息
  - 原始终端流日志
