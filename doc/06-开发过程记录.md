## 3. 最近一次推进记录

### 3.1 本轮核心目标

围绕“让会话详情页具备更接近真实终端的交互能力”推进，优先解决以下问题：

- 原始输出链路不完整，导致页内终端体验不稳定。
- 会话启动后缺少终端尺寸同步，TUI 类程序展示异常。
- 输入记录与初始化输入混在一起，影响消息记录语义。
- 不同运行环境下存在字符集差异，输出可能出现乱码或截断。

### 3.2 本轮已完成

#### 后端

- `SessionController` 新增会话原始输出查询接口：`GET /api/v1/sessions/{id}/raw-output`
- `SessionController` 新增终端尺寸调整接口：`POST /api/v1/sessions/{id}/terminal/resize`
- `SessionAppService` 新增原始日志读取能力
- `SessionAppService` 调整输入发送逻辑，支持 `recordInput` 控制是否记录用户输入
- `SessionAppService` 新增运行中会话的终端尺寸调整逻辑
- `LocalProcessSupervisor` 引入 `pty4j` 托管，支持：
  - 在 `embedded` 或 `RAW` 交互模式下使用 `PTY`
  - 初始终端尺寸设置
  - 运行中 `resize`
  - `stdout/stderr` 与 `PTY` 输出的统一处理
  - 基于 `UTF-8 / UTF-16 / fallback charset` 的分块解码

#### 前后端联动

根据上轮会话记录，前端已同步接入以下链路：

- 会话详情页在 `RAW` 或 `embedded` 场景下走完整的页内终端链路
- 前端 `xterm` 已根据容器尺寸变化触发后端 `resize`
- 前端调用后端新增原始输出接口，补齐页内终端恢复逻辑

> 注：前端代码位于 `D:\Project\ali\260409\mutiAgent-Web`，本仓库文档仅记录联动结果，不在此处保存前端实现细节。

## 8. 2026-04-09 第二次推进：历史检索 / 全文搜索首版落地

### 8.1 本轮新增能力

#### 后端

- 新增历史检索控制器：`/api/v1/history/search`
- 新增历史检索应用服务：`HistoryAppService`
- 新增历史检索数据访问层：`HistoryMapper`
- 支持按以下条件组合检索：
  - `keyword`
  - `appType`
  - `projectPath`
  - `dateFrom`
  - `dateTo`
- 搜索结果拆分为两类：
  - 会话命中
  - 消息命中
- 数据库补齐 `message_fts` 初始化回填逻辑
- 消息写入时同步写入 `message_fts`，保证新消息可以被检索

#### 前端

- 在会话管理页新增“历史检索 / 全文搜索”搜索卡片
- 支持关键词、应用类型、项目路径、日期范围筛选
- 搜索结果拆分展示为：
  - 会话命中表格
  - 消息命中表格
- 支持从搜索结果直接跳转回会话详情页

### 8.2 本轮实现说明

- SQLite 触发器方案在 Spring SQL 初始化阶段存在兼容问题，因此改为“代码侧同步 FTS 索引”
- 这样做的优点是：
  - 启动更稳定
  - 行为更可控
  - 便于后续针对消息更新、重建索引做显式治理

已验证：

  - 新增：

    - `journal_mode = WAL`
    - `synchronous = NORMAL`
    - `foreign_keys = ON`
  - 保持连接池 `maximumPoolSize = 1`

## 40. 2026-04-13 继续推进：历史搜索改为异步预热，避免首次搜索卡顿

### 40.1 本轮问题
