# 项目目录结构设计

## 1. 推荐项目根目录

建议以当前目录为基础，保持前后端分仓但同级协作：

```text
D:\Project\ali\260409
├─mutiAgent                 # Spring Boot 后端 + 项目文档
└─mutiAgent-Web             # Vue 3 + Electron 前端桌面壳
```

这样做的好处：

- 前后端职责分离，前端与桌面壳可以单独迭代
- 后端仍保持 Java 主体，不被 Electron 工程污染
- 部署时可以再打成一个桌面安装包，但开发时保持清晰

## 4. 目录职责说明

### 后端 `mutiAgent`

- `controller`
  - 暴露 REST / WebSocket 接口
  - 只做参数校验、鉴别命令对象、返回统一响应
- `application`
  - 编排用例，例如创建实例、启动会话、发送输入、停止会话
  - 不直接依赖具体 CLI
- `domain`
  - 放领域对象、仓储接口、状态机、核心规则
- `infrastructure`
  - 放 SQLite、MyBatis、PTY、外部进程、文件日志、AI 适配器实现
- `bootstrap`
  - 放启动时初始化逻辑，如数据库检查、默认配置生成、运行目录创建
- `scripts`
  - 放本地开发脚本、打包脚本、数据库初始化脚本

### 前端 `mutiAgent-Web`

- `electron/main`
  - Electron 主进程
  - 负责窗口、托盘、菜单、调用系统原生命令
- `electron/preload`
  - 对 Renderer 暴露安全 API
- `src/api`
  - 封装后端 REST / WebSocket 客户端
- `src/stores`
  - 存应用实例、会话列表、消息、运行态
- `src/views`
  - 页面级视图
- `src/components/terminal`
  - 放 `xterm.js` 相关组件和适配器

## 5. 本地运行目录建议

后端运行时不要把所有运行数据直接塞在源码目录里，建议启动时统一映射到：

```text
%LOCALAPPDATA%\mutiAgent
├─data
│  └─muti-agent.db
├─logs
│  ├─app
│  └─sessions
├─runtime
│  ├─pid
│  └─cache
└─config
   └─default-config.json
```

原因：

- 便于代码仓库保持干净
- 打包后也能复用同一运行目录
- 会话原始日志文件通常较大，不适合放源码目录

## 6. 前后端协作边界

推荐采用以下原则：

- Electron 负责：
  - 窗口、托盘、菜单、文件选择、本地系统调用、启动 Java 服务
- Spring Boot 负责：
  - 会话、实例、消息、数据库、适配器、状态机、历史记录
- Vue 页面负责：
  - 展示、交互、管理流、终端展示

这能避免出现“业务逻辑一半在 Electron，一半在后端”的失控问题。

## 8. 命名建议

- 后端包名统一用：`com.aliano.mutiagent`
- 前端项目名统一用：`muti-agent-console`
- 数据库文件名统一用：`muti-agent.db`
- WebSocket 事件名统一前缀：`session.*`
- Electron IPC 通道统一前缀：`desktop:*`
